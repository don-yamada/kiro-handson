## Kiroが応答しない

### 症状

- メッセージを送信しても、Kiroが応答しない
- 処理が途中で止まる
- 「考え中...」のまま進まない

### 原因と解決方法

#### 原因1: ネットワーク接続の問題

**確認方法**:
- インターネット接続を確認
- 他のWebサイトにアクセスできるか確認

**解決方法**:
```
1. Wi-Fiまたは有線接続を確認
2. ルーターを再起動
3. Kiroを再起動
```

#### 原因2: サーバーの一時的な問題

**解決方法**:
```
1. 数分待ってから再試行
2. Kiroを再起動
3. 別のメッセージを送信してみる
```

#### 原因3: 複雑すぎる指示

**解決方法**:
```
1. 指示をシンプルにする
2. 一度に1つのことだけを依頼する
3. 具体的で明確な指示を出す
```

例：
❌ 悪い指示:
```
プロジェクト全体を作成して、テストも書いて、ドキュメントも更新して
```

✅ 良い指示:
```
まず、src/app.jsを作成してください。
```

## クレジット不足

### 症状

- 「クレジットが不足しています」というメッセージが表示される
- Kiroが応答を停止する

### 確認方法

```
1. Kiroの右上のアカウントアイコンをクリック
2. 「Settings」→「Billing」を選択
3. 現在のクレジット残高を確認
```

### 解決方法

#### 方法1: 有料プランにアップグレード

```
1. 設定画面の「Billing」タブを開く
2. 「Upgrade」ボタンをクリック
3. プランを選択（Pro Tier または Pro+ Tier）
4. 支払い情報を入力
```

#### 方法2: クレジットを節約して使用

- 明確で具体的な指示を出す
- 不要な再生成を避ける
- AgentHooksは短時間のみ有効化

#### 方法3: 翌月まで待つ

有料プランの場合、翌月にクレジットがリセットされます。

## エラーメッセージの解釈

### よくあるエラーメッセージ

#### 「File not found」

**意味**: 指定したファイルが見つかりません

**解決方法**:
```
1. ファイルパスが正しいか確認
2. ファイルが存在するか確認
3. Kiroに「プロジェクトのファイル一覧を見せてください」と依頼
```

#### 「Permission denied」

**意味**: ファイルやディレクトリへのアクセス権限がありません

**解決方法**:
```
1. ファイルの権限を確認
2. 管理者権限で実行
3. ファイルが他のプログラムで開かれていないか確認
```

#### 「Syntax error」

**意味**: コードに構文エラーがあります

**解決方法**:
```
Kiroに「このファイルの構文エラーを修正してください」と依頼
```

#### 「Timeout」

**意味**: 処理がタイムアウトしました

**解決方法**:
```
1. 処理を小さな単位に分割
2. 再試行
3. ネットワーク接続を確認
```

## コード生成が期待通りでない

### 症状

- 生成されたコードが要件を満たしていない
- コードが複雑すぎる、またはシンプルすぎる
- エラーが含まれている

### 解決方法

#### 方法1: より具体的な指示を出す

❌ 悪い例:
```
Todoアプリを作って
```

✅ 良い例:
```
HTML、CSS、JavaScriptを使って、シンプルなTodoアプリを作成してください。
機能は以下の3つです：
1. Todoの追加
2. Todoの完了
3. Todoの削除
```

#### 方法2: 段階的に修正を依頼

```
このコードをもっとシンプルにしてください。
```

```
このコードに、初心者でも理解できるようにコメントを追加してください。
```

```
このコードのエラーを修正してください。
```

#### 方法3: 参考例を提示

```
以下のような形式でコードを生成してください：
[参考コード]
```

## ファイルが見つからない

### 症状

- レッスンで参照しているファイルが存在しない
- Kiroが「ファイルが見つかりません」と応答する

### 解決方法

#### 方法1: ファイル構造を確認

```
Kiroに「プロジェクトのファイル構造を見せてください」と依頼
```

#### 方法2: ファイルを作成

```
Kiroに「[ファイルパス]を作成してください」と依頼
```

#### 方法3: パスを確認

- 相対パスが正しいか確認
- ファイル名のスペルミスがないか確認
- 大文字小文字が正しいか確認（macOS/Linuxは区別する）

## AgentHooksが動作しない

### 症状

- Hooksを設定したが、自動実行されない
- エラーメッセージが表示される

### 解決方法

#### 方法1: 設定を確認

```
Kiroに「AgentHooksの設定を見せてください」と依頼
```

確認ポイント：
- イベントトリガーが正しいか
- ファイルパターンが正しいか
- Hooksが有効になっているか

#### 方法2: Hooksを再作成

```
1. 既存のHooksを削除
2. もう一度Hooksを作成
3. 動作を確認
```

#### 方法3: クレジット残高を確認

AgentHooksはクレジットを消費します。残高が不足していないか確認してください。

#### 方法4: ファイルパターンを確認

```
悪い例: *.js（カレントディレクトリのみ）
良い例: **/*.js（すべてのサブディレクトリを含む）
```

## ステアリングファイルが反映されない

### 症状

- ステアリングファイルを編集したが、Kiroの応答が変わらない
- 設定した性格や語尾が反映されない

### 解決方法

#### 方法1: Kiroに読み込み直しを依頼

```
ステアリングファイルの内容を読み込み直してください。
```

#### 方法2: プロジェクトを開き直す

```
1. Kiroでプロジェクトを閉じる
2. もう一度プロジェクトを開く
3. 動作を確認
```

#### 方法3: ファイルの場所を確認

ステアリングファイルは `.kiro/steering.md` に配置されている必要があります。

#### 方法4: ファイルの内容を確認

```
Kiroに「.kiro/steering.mdの内容を見せてください」と依頼
```

## MCPサーバーが動作しない

### 症状

- MCPサーバーに接続できない
- 「Server not found」エラーが表示される

### 解決方法

#### 方法1: MCPサーバーの設定を確認

```
Kiroに「.kiro/settings/mcp.jsonの内容を見せてください」と依頼
```

確認ポイント：
- サーバーが有効になっているか（disabled: false）
- コマンドとパスが正しいか
- 環境変数が設定されているか

#### 方法2: 必要なツールがインストールされているか確認

例：
- `uvx` コマンドが使用可能か
- Python がインストールされているか
- 必要なパッケージがインストールされているか

#### 方法3: MCPサーバーを再起動

```
1. Kiroを再起動
2. MCPサーバーが自動的に再接続される
```

#### 方法4: ログを確認

```
Kiroに「MCPサーバーのログを見せてください」と依頼
```

## 段階的なトラブルシューティング手順

問題が解決しない場合は、以下の手順を試してください：

### ステップ1: 基本的な確認

1. インターネット接続を確認
2. Kiroを最新バージョンに更新
3. クレジット残高を確認
4. Kiroを再起動

### ステップ2: 問題の特定

1. エラーメッセージを記録
2. 問題が発生する手順を再現
3. 他のプロジェクトでも同じ問題が発生するか確認

### ステップ3: 情報収集

1. Kiroの公式ドキュメントを確認
2. コミュニティで同様の問題を検索
3. エラーメッセージでWeb検索

### ステップ4: サポートに問い合わせ

上記の方法で解決しない場合は、サポートに問い合わせてください。

## よくある質問（FAQ）

### Q: トライアルクレジットはいつまで有効？

A: トライアルクレジットには通常、有効期限があります。詳細はKiroの設定画面で確認してください。

### Q: クレジットの消費量を確認する方法は？

A: 設定画面の「Billing」タブで、使用履歴とクレジット残高を確認できます。

### Q: 有料プランはいつでもキャンセルできる？

A: はい、いつでもキャンセル可能です。キャンセル後も、当月分は利用できます。

### Q: チームで使う場合の料金は？

A: ユーザーごとにアカウントが必要です。チーム向けプランについては、Enterpriseプランをご検討ください。

### Q: オフラインで使える？

A: Kiroはクラウドベースのサービスのため、インターネット接続が必要です。

### Q: データのプライバシーは？

A: Kiroは業界標準のセキュリティ対策を実施しています。詳細は[公式サイト](https://kiro.dev/)をご確認ください。

## 緊急時の対処法

### Kiroが全く動作しない場合

1. **Kiroを完全に終了**
   - macOS: Cmd+Q
   - Windows: Alt+F4
   - Linux: Ctrl+Q

2. **Kiroを再起動**

3. **それでも動作しない場合**
   - Kiroをアンインストール
   - 最新版を再インストール
   - [Kiro公式サイト](https://kiro.dev/)からダウンロード

### データが失われた場合

- Kiroは自動保存機能があります
- プロジェクトフォルダを確認
- Gitを使用している場合は、コミット履歴を確認
