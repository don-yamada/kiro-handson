## 📖 AgentHooksとは？

### 概要

AgentHooksは、特定のイベント（ファイル保存、コミットなど）が発生したときに、Kiroが自動的に処理を実行する機能です。

### できること

- **ファイル保存時**: 自動でコードをフォーマット、テストを実行
- **コミット時**: 自動でコードレビュー、ドキュメント更新
- **プロンプト送信時**: 特定のルールを自動適用
- **ツール使用前後**: 検証や後処理を自動実行

### 基本機能との違い

| 項目 | 基本機能 | AgentHooks |
|------|---------|-----------|
| 実行方法 | 手動で指示 | 自動実行 |
| クレジット消費 | 指示ごと | イベントごと |
| 推奨プラン | トライアル可 | Pro Tier以上 |
| 用途 | 個別の作業 | 継続的な自動化 |

## イベントトリガーの種類

AgentHooksで使用できる主なイベントトリガー：

### ファイル関連

- **fileEdited**: ファイルが編集されたとき
- **fileCreated**: ファイルが作成されたとき
- **fileDeleted**: ファイルが削除されたとき

### プロンプト関連

- **promptSubmit**: メッセージを送信したとき
- **agentStop**: Kiroの処理が完了したとき

### ツール関連

- **preToolUse**: ツール使用前
- **postToolUse**: ツール使用後

### 手動トリガー

- **userTriggered**: ユーザーが手動で実行

## クレジット消費の注意点

### 消費パターン

AgentHooksは、イベントが発生するたびにクレジットを消費します：

- ファイル保存1回 = 約5-10クレジット
- 複数ファイルを連続保存 = 消費が積み重なる
- 自動フォーマット + テスト実行 = さらに消費

### 節約のコツ

1. **必要なときだけ有効化**: 開発中のみHooksを有効にする
2. **対象ファイルを絞る**: すべてのファイルではなく、特定の拡張子のみ
3. **シンプルな処理**: 複雑な処理は避ける
4. **手動トリガーを活用**: 自動実行ではなく、必要なときだけ実行

### 有料プランの必要性

AgentHooksを実運用で使用する場合：

- **Pro Tier（約$20/月）**: 月間1,000クレジット
  - 1日あたり約30回のHook実行が可能
  - 個人開発に適している

- **Pro+ Tier（約$40/月）**: 月間2,000-3,000クレジット
  - 1日あたり約60-100回のHook実行が可能
  - チーム開発に適している

## 💡 よくある質問

**Q: Hooksが動作しない**
A: Hookの設定ファイルを確認してください。イベントトリガーとファイルパターンが正しいか確認しましょう。

**Q: クレジットを大量に消費してしまった**
A: すぐにHooksを無効化してください。今後は、短時間だけ有効化して、すぐに無効化することをおすすめします。

**Q: どんなHooksが実用的か**
A: コードフォーマット、テスト実行、ドキュメント更新などが人気です。ただし、実運用には有料プランが必要です。

**Q: トライアルクレジットで十分試せる？**
A: 基本的な動作確認は可能ですが、実運用には有料プランをおすすめします。
