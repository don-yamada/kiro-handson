# レッスン2: コード生成とリファクタリング

レッスン1で作成したSpecに基づいて、実際にコードを生成してみましょう！このレッスンでは、Kiroのコード生成機能、リファクタリング機能、ファイル操作を体験します。

## 🎯 学習目標

このレッスンを完了すると、以下のことができるようになります：

1. **タスクベースの開発**: Tasks.mdに基づいてコードを生成
2. **コード生成**: Kiroに指示してコードを自動生成
3. **リファクタリング**: 既存のコードを改善
4. **ファイル操作**: ファイルの作成、編集、確認
5. **検索機能**: プロジェクト内のコードを検索

## ⏱️ 所要時間

約30分

## 📚 前提知識

- レッスン1を完了していること
- Specファイル（Requirements、Design、Tasks）が作成されていること

## 📖 レッスンの流れ

### ステップ1: Tasks.mdを確認する

まず、レッスン1で作成したタスクリストを確認しましょう。

Kiroに以下のように話しかけてください：

```
.kiro/specs/[プロジェクト名]/tasks.mdを開いて、最初のタスクを教えてください。
```

Tasks.mdには、実装すべきタスクが順番に並んでいます。各タスクには以下の情報が含まれています：

- タスクの内容
- 対応する要件番号
- サブタスク（ある場合）

### ステップ2: 最初のタスクを実行する

Kiroに最初のタスクを実行してもらいましょう。

**基本的な指示**:
```
tasks.mdの最初のタスクを実行してください。
```

**より具体的な指示**:
```
tasks.mdのタスク1を実行してください。
必要なファイルを作成し、基本的な構造を実装してください。
```

Kiroが以下のことを自動的に行います：

1. 必要なファイルを作成
2. 基本的なコードを生成
3. コメントや説明を追加
4. タスクの完了を報告

### ステップ3: 生成されたコードを確認する

Kiroが生成したコードを確認しましょう。

```
生成されたファイルの一覧を見せてください。
```

```
[ファイル名]を開いて、内容を説明してください。
```

💡 **確認ポイント**:
- ファイルが正しい場所に作成されているか
- コードが要件を満たしているか
- コメントがわかりやすいか
- 初心者でも理解できる実装か

### ステップ4: 次のタスクを実行する

同じように、次のタスクも実行していきましょう。

```
tasks.mdの次のタスクを実行してください。
```

または

```
tasks.mdのタスク2を実行してください。
```

**時間の目安**:
- このレッスンでは、2-3個のタスクを実行することをおすすめします
- すべてのタスクを完了する必要はありません（レッスン2の目的は、コード生成の流れを体験することです）

## リファクタリング機能を試す

生成されたコードを改善してみましょう。

### パターン1: コードの可読性を向上

```
[ファイル名]の[関数名]をリファクタリングして、より読みやすくしてください。
```

例：
```
src/app.jsのaddTask関数をリファクタリングして、より読みやすくしてください。
```

### パターン2: 関数を分割

```
[ファイル名]の[関数名]が長すぎるので、複数の小さな関数に分割してください。
```

### パターン3: 変数名を改善

```
[ファイル名]の変数名をより説明的な名前に変更してください。
```

### パターン4: コメントを追加

```
[ファイル名]に、初心者でも理解できるように詳しいコメントを追加してください。
```

## ファイル操作機能を試す

### 新しいファイルを作成

```
src/utils.jsファイルを作成して、[機能]を実装してください。
```

例：
```
src/utils.jsファイルを作成して、日付をフォーマットする関数を実装してください。
```

### ファイルの内容を確認

```
[ファイル名]の内容を見せてください。
```

### ファイルを編集

```
[ファイル名]に[機能]を追加してください。
```

例：
```
src/app.jsにエラーハンドリングを追加してください。
```

## 💡 よくある質問

**Q: タスクの実行に失敗する**
A: タスクの内容が複雑すぎる可能性があります。「もっとシンプルに実装してください」と指示してみてください。

**Q: 生成されたコードが動かない**
A: 「このコードにエラーがないか確認して、修正してください」と指示してください。

**Q: リファクタリングで何が変わったかわからない**
A: 「リファクタリング前後の違いを説明してください」と聞いてみましょう。

**Q: どのタスクから始めればいいかわからない**
A: Tasks.mdの順番通りに進めるのがおすすめです。Kiroに「次に実行すべきタスクを教えてください」と聞いてもOKです。

**Q: クレジットを節約したい**
A: 一度に複数のタスクをまとめて実行するよりも、1つずつ確認しながら進める方が、結果的に再生成が少なくなります。
