# レッスン1: 指示例集

このファイルには、レッスン1で使用する具体的なKiroへの指示例をまとめています。コピー＆ペーストして使用できます。

## パート1: 初期設定 - ステアリングファイルのカスタマイズ

### ステップ1: ステアリングファイルを開く

```
.kiro/steering.mdを開いて、内容を見せてください。
```

### ステップ2: Kiroの性格を変更する

以下から好きなパターンを選んで、Kiroに指示してください：

#### パターン1: カジュアルな友達風

```
ステアリングファイルを編集して、Kiroの性格を「カジュアルで親しみやすい友達」に、
語尾を「〜だよ」「〜だね」に変更してください。
```

#### パターン2: 丁寧な先生風

```
ステアリングファイルを編集して、Kiroの性格を「丁寧で教育的な先生」に、
語尾を「〜ます」「〜ですね」に変更してください。
```

#### パターン3: 上品なお嬢様風

```
ステアリングファイルを編集して、Kiroの性格を「上品で優雅なお嬢様」に、
語尾を「〜ですわ」「〜ですの」に変更してください。
```

#### パターン4: 元気なアイドル風

```
ステアリングファイルを編集して、Kiroの性格を「元気で明るいアイドル」に、
語尾を「〜だよ！」「〜だね☆」に変更してください。
```

#### パターン5: クールな専門家風

```
ステアリングファイルを編集して、Kiroの性格を「冷静で論理的な専門家」に、
語尾を「〜である」「〜だ」に変更してください。
```

#### カスタムパターン

自分で考えた性格を設定することもできます：

```
ステアリングファイルを編集して、Kiroの性格を「[あなたの考えた性格]」に、
語尾を「[あなたの考えた語尾]」に変更してください。
```

例：
- 「関西弁を話すお笑い芸人」「〜やで」「〜やん」
- 「武士のような古風な話し方」「〜でござる」「〜じゃ」
- 「ロボットのような機械的な話し方」「〜デス」「〜マス」

### ステップ3: 変化を確認する

```
こんにちは！今日の調子はどう？このプロジェクトについて教えて。
```

または

```
自己紹介してください。あなたの性格や特徴を教えてください。
```

## パート2: Spec作成 - Requirements-first Workflow

### ステップ1: プロジェクトのアイデアを伝える

以下から好きなテーマを選んで、Kiroに指示してください：

#### テーマ1: シンプルなTodoアプリ

```
シンプルなTodoアプリを作りたいです。
タスクの追加、完了、削除ができる機能が欲しいです。
Requirements-first Workflowで進めてください。
```

#### テーマ2: 名言ジェネレーター

```
ランダムに名言を表示するアプリを作りたいです。
ボタンを押すと新しい名言が表示される機能が欲しいです。
Requirements-first Workflowで進めてください。
```

#### テーマ3: 簡単な計算機

```
四則演算ができる計算機アプリを作りたいです。
足し算、引き算、掛け算、割り算ができる機能が欲しいです。
Requirements-first Workflowで進めてください。
```

#### テーマ4: タイマーアプリ

```
カウントダウンタイマーアプリを作りたいです。
時間を設定して、カウントダウンを開始・停止・リセットできる機能が欲しいです。
Requirements-first Workflowで進めてください。
```

#### テーマ5: じゃんけんゲーム

```
コンピュータと対戦するじゃんけんゲームを作りたいです。
グー・チョキ・パーを選んで、勝敗を判定する機能が欲しいです。
Requirements-first Workflowで進めてください。
```

#### カスタムテーマ

自分でアイデアを考えることもできます：

```
[あなたのアイデア]を作りたいです。
[必要な機能]が欲しいです。
Requirements-first Workflowで進めてください。
```

💡 **ヒント**: 
- プログラミング初心者でも理解できるシンプルなものがおすすめ
- 機能は3-5個程度に絞ると、30分以内で完了しやすい
- Webアプリ、CLIツール、デスクトップアプリなど、形式は自由

### ステップ2: Kiroの質問に答える

Kiroが質問してきたら、以下のように答えることができます：

#### 対象ユーザーについて聞かれたら

```
プログラミング初心者を含む一般ユーザーを対象にしています。
```

#### プラットフォームについて聞かれたら

```
Webブラウザで動くアプリにしてください。
```

または

```
おすすめのプラットフォームを教えてください。
```

#### データ保存について聞かれたら

```
ブラウザのローカルストレージに保存してください。
```

または

```
シンプルな方法を提案してください。
```

#### 技術スタックについて聞かれたら

```
HTML、CSS、JavaScriptで作成してください。
```

または

```
初心者でも理解しやすい技術を選んでください。
```

#### わからないことがあったら

```
おすすめを教えてください。
```

または

```
初心者向けの選択肢を提案してください。
```

### ステップ3: 生成された設計書を確認する

#### Requirements.mdを確認

```
.kiro/specs/[プロジェクト名]/requirements.mdを開いて、内容を説明してください。
```

#### Design.mdを確認

```
.kiro/specs/[プロジェクト名]/design.mdを開いて、主要なコンポーネントを説明してください。
```

#### Tasks.mdを確認

```
.kiro/specs/[プロジェクト名]/tasks.mdを開いて、最初のタスクを教えてください。
```

### ステップ4: 設計書を修正する（必要に応じて）

#### 要件を追加したい場合

```
requirements.mdに「[追加したい機能]」の要件を追加してください。
```

例：
```
requirements.mdに「ダークモード対応」の要件を追加してください。
```

#### 設計を変更したい場合

```
design.mdの[変更したい部分]を[どう変更するか]に変更してください。
```

例：
```
design.mdのデータ保存方法をローカルストレージからIndexedDBに変更してください。
```

#### タスクの順序を変更したい場合

```
tasks.mdのタスク[番号]と[番号]の順序を入れ替えてください。
```

## 💡 よくある質問と対処法

### Q: ステアリングファイルの変更が反映されない

**対処法1: Kiroに確認を依頼**
```
ステアリングファイルの内容を読み込み直してください。
```

**対処法2: プロジェクトを開き直す**
Kiroでプロジェクトを閉じて、再度開いてください。

### Q: Spec作成に時間がかかりすぎる

**対処法: シンプルなアイデアから始める**
```
もっとシンプルなバージョンで、最小限の機能だけに絞ってください。
```

### Q: 生成された設計書が期待と違う

**対処法: 具体的に修正を依頼**
```
requirements.mdの[具体的な箇所]を[どう変更するか]に修正してください。
```

### Q: どのテーマを選べばいいかわからない

**対処法: Kiroに相談**
```
プログラミング初心者が30分で完成できる、おすすめのプロジェクトテーマを3つ提案してください。
```

### Q: Kiroの質問の意味がわからない

**対処法: 説明を求める**
```
その質問の意味を、初心者にもわかりやすく説明してください。
```
