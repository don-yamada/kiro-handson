## 📖 MCPとは？

### 概要

MCP（Model Context Protocol）は、Kiroが外部のツールやサービスと連携するための標準プロトコルです。

### できること

- **ファイルシステム**: ローカルファイルの読み書き
- **データベース**: SQLクエリの実行、データ取得
- **Web API**: 外部サービスとの連携
- **開発ツール**: Git、Docker、テストツールなど
- **カスタムツール**: 自社システムとの統合

### 基本機能との違い

| 項目 | 基本機能 | MCP連携 |
|------|---------|---------|
| 対象 | Kiroの標準機能 | 外部ツール・サービス |
| 設定 | 不要 | MCPサーバーの設定が必要 |
| 用途 | 一般的な開発作業 | 特定のツール・サービス連携 |
| 拡張性 | 固定 | カスタマイズ可能 |

### ステップ4: MCP連携の設定を確認する

MCP連携の設定ファイルを確認しましょう。

```
.kiro/settings/mcp.jsonを開いて、内容を見せてください。
```

設定ファイルには、以下の情報が含まれています：

- **mcpServers**: 利用可能なMCPサーバーの一覧
- **command**: サーバーの起動コマンド
- **args**: コマンドライン引数
- **env**: 環境変数
- **disabled**: 有効/無効の設定

## MCPサーバーの種類

### 開発ツール系

- **PostgreSQL**: SQLクエリ実行、データ取得

- **MongoDB**: NoSQLデータベース操作

### 外部サービス系

- **GitHub**: リポジトリ、Issue、PR操作
- **Slack**: メッセージ送信、チャンネル管理
- **AWS**: クラウドリソース操作

### ユーティリティ系

- **filesystem**: ファイル操作
- **web-search**: Web検索
- **puppeteer**: ブラウザ自動操作

## エンタープライズ向け機能

### 自社システムとの連携

MCPを使って、自社のシステムやデータベースと連携できます：

- **社内データベース**: 顧客情報、在庫情報の取得
- **社内API**: 業務システムとの連携
- **カスタムツール**: 独自開発したツールの統合

### セキュリティとアクセス制御

- **認証**: APIキー、OAuth、SSO
- **権限管理**: ユーザーごとのアクセス制御
- **監査ログ**: 操作履歴の記録

### スケーラビリティ

- **専用サーバー**: 高負荷に対応
- **負荷分散**: 複数サーバーでの分散処理
- **キャッシング**: 応答速度の向上
